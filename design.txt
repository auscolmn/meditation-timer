Sati - Design System Implementation Instructions
Instructions for Claude Code
Update the Sati meditation timer app with the following design system:
1. Install Inter Font
Add to the <head> section of index.html or at the top of your main CSS file:
html<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500&display=swap" rel="stylesheet">
2. Update CSS Variables - Subtle Earth Color Palette
Replace existing color variables with:
css:root {
  /* Primary Colors */
  --primary: #3D3D3D;           /* Charcoal - main interactive elements */
  --primary-light: #6B6B6B;     /* Light charcoal - hover states */
  --primary-lighter: #9A9A9A;   /* Lighter - disabled states */

  /* Success/Completion */
  --success: #5C7A5C;           /* Muted sage green - streaks, completed sessions */
  --success-light: #7A9A7A;     /* Light sage - success backgrounds */
  --success-bg: #F0F4F0;        /* Very light sage - subtle backgrounds */

  /* Backgrounds */
  --background: #F5F5F0;        /* Warm off-white - main background */
  --surface: #FAFAF8;           /* Warm white - cards, surfaces */
  --surface-raised: #FFFFFF;    /* Pure white - elevated elements */

  /* Text */
  --text-primary: #2B2B2B;      /* Almost black - main text */
  --text-secondary: #7A7A7A;    /* Medium gray - secondary text */
  --text-tertiary: #ABABAB;     /* Light gray - placeholder text */

  /* Borders & Dividers */
  --border: #E5E5E0;            /* Subtle warm border */
  --border-light: #F0F0EB;      /* Very subtle border */

  /* Shadows (very subtle) */
  --shadow-sm: 0 1px 2px rgba(0, 0, 0, 0.04);
  --shadow-md: 0 2px 8px rgba(0, 0, 0, 0.06);
  --shadow-lg: 0 4px 16px rgba(0, 0, 0, 0.08);

  /* Spacing Scale (generous for breathing room) */
  --space-xs: 4px;
  --space-sm: 8px;
  --space-md: 16px;
  --space-lg: 24px;
  --space-xl: 32px;
  --space-2xl: 48px;
  --space-3xl: 64px;

  /* Border Radius (subtle, organic) */
  --radius-sm: 4px;
  --radius-md: 8px;
  --radius-lg: 12px;
  --radius-xl: 16px;
  --radius-full: 9999px;

  /* Transitions */
  --transition-fast: 150ms ease;
  --transition-base: 200ms ease;
  --transition-slow: 300ms ease;
}

4. Apply Design System Throughout App
Key Mapping:

All green/completion colors → var(--success)
All background colors → var(--background) or var(--surface)
All buttons → Update to use new color variables
All text → Use var(--text-primary), var(--text-secondary), or var(--text-tertiary)
All borders → Use var(--border) or var(--border-light)
All shadows → Use var(--shadow-sm), var(--shadow-md), or var(--shadow-lg)
All spacing → Use var(--space-*) scale
All border-radius → Use var(--radius-*) scale

Specific Components:

Completed session dots in calendar → var(--success)
Streak indicators → var(--success)
Progress circles/bars → var(--success)
Cards/surfaces → var(--surface) background with var(--shadow-sm)
Input fields → var(--surface-raised) background, var(--border) border
Active navigation tab → var(--primary)
Inactive navigation tab → var(--text-secondary)

5. Design Principles to Follow
Whitespace:

Use generous spacing: var(--space-xl) and var(--space-2xl) between major sections
Padding inside cards: var(--space-xl)
Margin between elements: Minimum var(--space-md)

Minimal Visual Noise:

Prefer spacing over borders
Use very subtle shadows only
Keep borders light: var(--border-light) where possible
Avoid bold colors except for success states

Typography:

Only use font weights 400 and 500 (no bold/700)
Apply negative letter-spacing for cleaner look
Use tabular nums for timer display

Interactive Elements:

Minimum tap target: 44x44px
Smooth transitions: var(--transition-base)
Subtle hover effects (don't add heavy shadows or bright colors)
